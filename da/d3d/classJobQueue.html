Detailed Description

A thread-safe FIFO queue for job dispatching.

This queue is intended for use in thread pools and other concurrent job-processing systems. It provides deterministic shutdown semantics and safe synchronization between multiple producers and consumers.

Queue states:

  • Open: accepts new jobs; pop() may block.
  • Closed: rejects no jobs explicitly, but pop() returns nullptr once the queue drains.

◆ pop()

std::unique_ptr< IJob > JobQueue::pop ( )

Pops the next available job (blocking).

Retrieves the next available job, blocking if necessary.

Returns
A std::unique_ptr<IJob> containing the next job, or nullptr if the queue has been closed and no jobs remain.
  • Blocks while the queue is empty and still open.
  • If the queue is closed and empty, returns nullptr immediately.
Returns
  • A std::unique_ptr<IJob> containing the next job.
  • nullptr if the queue is closed AND empty.

Blocking behaviour:

  • If the queue is empty and not closed, the caller blocks.
  • The thread wakes when:
    1. A new job is pushed, or
    2. shutdown() is called.

Shutdown semantics:

  • If closed == true and there are no jobs remaining, this function returns nullptr immediately.

Concurrency:

  • buffer is protected by mtx.
  • Uses a condition-variable predicate (closed || !buffer.empty()), which prevents spurious wakeups causing incorrect behaviour.

◆ shutdown()

void JobQueue::shutdown ( )

Closes the queue and wakes all waiting threads.

Once closed:

  • Consumers eventually return nullptr from pop().
  • The queue will not block again.

Effects:

  • Sets closed = true.
  • Wakes all threads waiting on pop().
  • Allows pop() to return nullptr when the queue becomes empty.

Concurrency:

  • Fully thread-safe; multiple calls are safe (idempotent).
