Detailed Description

A fixed-size pool of worker threads executing asynchronous jobs.

The pool starts N worker threads that repeatedly:

  1. Block on the shared JobQueue.
  2. Retrieve the next job.
  3. Execute it inside a try/catch to prevent worker termination.

ThreadPool is non-copyable and non-movable to avoid transferring thread ownership. Lifetime is strictly controlled: destruction forces a complete shutdown.

◆ ThreadPool() [2/3]

ThreadPool::ThreadPool ( const ThreadPool &  )
delete

Deleted copy constructor.

ThreadPool cannot be copied because it manages active threads and synchronization primitives. Copying would imply duplicating ownership of threads, which is undefined behavior.

◆ ThreadPool() [3/3]

ThreadPool::ThreadPool ( ThreadPool &&  )
delete

Deleted move constructor.

Although std::thread supports move semantics, allowing ThreadPool to be movable would transfer ownership of active threads and invalidate the source object. Movement is disabled to enforce strict lifetime control.

◆ shutdown()

void ThreadPool::shutdown ( )

Gracefully shuts down the pool.

  • Stops accepting new jobs.
  • Waits up to 1 second for the queue to drain.
  • Closes the queue.
  • Joins all workers.

Workers finish any job already in progress.

◆ shutdownNow()

void ThreadPool::shutdownNow ( )

Immediately shuts down the pool.

Immediately stops all workers, discarding queued jobs.

  • Stops accepting new jobs.
  • Immediately closes the queue.
  • Workers unblock and exit even if jobs remain unprocessed.

This is the “fail-fast” version of shutdown().
